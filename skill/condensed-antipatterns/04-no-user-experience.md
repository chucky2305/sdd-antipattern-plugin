## AP-04: Not connecting features into a user story

**What it means:** A list of features is not a user experience. You can build every feature and still ship something confusing because nobody described how they connect.

**What a good spec includes:**
- At least one end-to-end user journey written in plain language: "User opens the app. They see X. They do Y. They get Z."
- The journey is concrete and sequential, not abstract
